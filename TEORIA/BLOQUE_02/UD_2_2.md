## üìö Explicaci√≥n te√≥rica

Hasta ahora vimos funciones que no recib√≠an datos. Pero en la pr√°ctica, la mayor√≠a de las funciones trabajan con **informaci√≥n que reciben del exterior**.

üîë Conceptos clave:

* **Par√°metros** ‚Üí variables que se definen en la funci√≥n y que reciben datos.
* **Argumentos** ‚Üí los valores reales que pasamos cuando llamamos a la funci√≥n.

Ejemplo sencillo:

```python
def saludar(nombre):
    """Muestra un saludo usando un nombre recibido como par√°metro."""
    print(f"Hola {nombre}!")

saludar("Ana")  # 'Ana' es el argumento
```

üëâ Esto hace que las funciones sean **reutilizables y personalizables**.

---

### üìå Tipos de par√°metros

1. **Par√°metros posicionales**

   * Se pasan en el mismo orden en que se definieron.

   ```python
   def saludar(nombre, edad):
       print(f"Hola {nombre}, tienes {edad} a√±os.")

   saludar("Ana", 20)  # nombre="Ana", edad=20
   ```

---

2. **Par√°metros con valor por defecto**

   * Si no se pasa un argumento, se usa el valor establecido por defecto.

   ```python
   def presentar(nombre, pais="Espa√±a"):
       print(f"Me llamo {nombre} y soy de {pais}.")

   presentar("Carlos")            # usa "Espa√±a" por defecto
   presentar("Laura", "M√©xico")   # sobrescribe el valor
   ```

---

3. **Argumentos nombrados (keyword arguments)**

   * Se indica el nombre del par√°metro al llamar a la funci√≥n.
   * Permiten cambiar el orden de los argumentos.

   ```python
   def info_usuario(nombre, edad, ciudad):
       print(f"{nombre}, {edad} a√±os, vive en {ciudad}.")

   info_usuario(ciudad="Madrid", edad=30, nombre="Luis")
   ```

---

‚úÖ **Buenas pr√°cticas con par√°metros:**

* Usa **nombres descriptivos** en los par√°metros (`nombre`, `edad`, `ciudad`) en lugar de letras sin sentido (`x`, `y`).
* Si una funci√≥n tiene muchos par√°metros, es recomendable usar **argumentos nombrados** para evitar errores de orden.
* Evita usar valores por defecto que sean **listas o diccionarios mutables**, ya que pueden causar errores inesperados.
