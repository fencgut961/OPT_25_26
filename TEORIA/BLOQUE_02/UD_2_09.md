## 1Ô∏è‚É£ ¬øQu√© es `zip()`?

`zip()` es una funci√≥n de Python que **une varias listas o colecciones** y te devuelve pares de elementos que ocupan la misma posici√≥n.

> Piensa en `zip()` como una **cremallera**: une los elementos de varias listas ‚Äúpor posici√≥n‚Äù.

---

### Ejemplo b√°sico

```python
numeros = [1, 2]
letras = ["A", "B"]

resultado = zip(numeros, letras)
print(list(resultado))
```

‚û°Ô∏è Resultado:

```
[(1, 'A'), (2, 'B')]
```

**Explicaci√≥n:**

* El primer par `(1, 'A')` combina el primer elemento de cada lista.
* El segundo par `(2, 'B')` combina el segundo elemento de cada lista.

---

## 2Ô∏è‚É£ Recorrer listas con `zip()`

`zip()` es muy √∫til junto con un `for` para recorrer varias listas **al mismo tiempo**:

```python
numeros = [1, 2]
letras = ["A", "B"]

for numero, letra in zip(numeros, letras):
    print("N√∫mero:", numero, "Letra:", letra)
```

‚û°Ô∏è Resultado:

```
N√∫mero: 1 Letra: A
N√∫mero: 2 Letra: B
```

**Explicaci√≥n paso a paso:**

1. En la primera vuelta del `for`, `numero` = 1 y `letra` = "A".
2. En la segunda vuelta, `numero` = 2 y `letra` = "B".
3. As√≠, recorremos las dos listas al mismo tiempo.

## 3Ô∏è‚É£ `zip()` con m√°s de dos listas

`zip()` no se limita a dos listas; podemos combinar **tres, cuatro o m√°s**.

```python
numeros = [1, 2]
espanol = ["Uno", "Dos"]
ingles = ["One", "Two"]

for n, e, i in zip(numeros, espanol, ingles):
    print(n, e, i)
```

‚û°Ô∏è Resultado:

```
1 Uno One
2 Dos Two
```

**Explicaci√≥n:**

* Cada vuelta del `for` toma los elementos que est√°n en la misma posici√≥n en todas las listas.

---

## 4Ô∏è‚É£ `zip()` con listas de diferente tama√±o

Si las listas tienen distinta cantidad de elementos, `zip()` **solo recorrer√° hasta donde alcance la lista m√°s corta**:

```python
numeros = [1, 2, 3]
letras = ["A", "B"]

for n, l in zip(numeros, letras):
    print(n, l)
```

‚û°Ô∏è Resultado:

```
1 A
2 B
```

**Explicaci√≥n:**

* El tercer elemento de `numeros` no se usa porque no hay un elemento correspondiente en `letras`.
* Esto evita errores y hace que siempre se emparejen los elementos correctamente.

---

## 5Ô∏è‚É£ `zip()` con diccionarios

Cuando usamos diccionarios, `zip()` recorre **las claves por defecto**:

```python
esp = {'1': 'Uno', '2': 'Dos'}
eng = {'1': 'One', '2': 'Two'}

for a, b in zip(esp, eng):
    print(a, b)
```

‚û°Ô∏è Resultado:

```
1 1
2 2
```

**Si queremos combinar claves y valores**:

```python
for (k1, v1), (k2, v2) in zip(esp.items(), eng.items()):
    print(k1, v1, v2)
```

‚û°Ô∏è Resultado:

```
1 Uno One
2 Dos Two
```

---

## 6Ô∏è‚É£ Deshacer un `zip()` (unpacking)

Si tenemos una lista de tuplas creada con `zip()`, podemos **separarla de nuevo**:

```python
c = [(1, 'A'), (2, 'B')]
a, b = zip(*c)

print(a)  # (1, 2)
print(b)  # ('A', 'B')
```

**Explicaci√≥n:**

* El operador `*` toma todas las tuplas y las separa en listas de elementos por posici√≥n.
* Esto nos permite recuperar las listas originales.
